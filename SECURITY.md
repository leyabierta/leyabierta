# Política de seguridad

## Reportar una vulnerabilidad

Si descubres una vulnerabilidad de seguridad en Ley Abierta, te pedimos que la reportes de forma responsable. **No abras un issue público.**

### Cómo reportar

1. **GitHub Private Vulnerability Reporting** (preferido): Usa la pestaña [Security](../../security/advisories/new) del repositorio para crear un informe privado. Solo los mantenedores lo verán.

2. **Correo electrónico**: Envía los detalles a **contacto@leyabierta.es** con el asunto "Vulnerabilidad de seguridad".

### Qué incluir

- Descripción de la vulnerabilidad
- Pasos para reproducirla
- Impacto potencial
- Si tienes una posible solución, inclúyela

### Qué esperar

- Confirmaremos la recepción en un máximo de **48 horas**
- Investigaremos y te mantendremos informado del progreso
- Publicaremos un fix y te daremos crédito (si lo deseas) una vez resuelto

### Alcance

Esta política cubre:

- El código de este repositorio (pipeline, API, web)
- La infraestructura de [leyabierta.es](https://leyabierta.es)
- La API pública

No cubre el contenido legislativo en sí (es dominio público procedente del BOE).

## Versiones soportadas

Únicamente la última versión desplegada en producción recibe actualizaciones de seguridad.
